Admin Service API Reference

Provides admin-panel operations including dashboard statistics, user management (roles, bans), and proxy access to article, subscription, promo code, and notification management. All routes require an admin-role JWT.

All RPCs are exposed through the GraphQL gateway. Every request requires access_token — a valid admin JWT.
GetDashboardStats admin

Returns aggregated platform statistics for the admin dashboard — user count, article count, views, and ban totals.

Request
FieldTypeRequired
access_tokenstringyes
Response
FieldTypeDescription
successbool
total_usersintTotal registered users
total_articlesintTotal published articles
total_categoriesintTotal article categories
total_viewsintTotal article views
articles_todayintArticles created today
users_todayintUsers registered today
total_bannedintTotal banned users
json
// Request
{ "access_token": "eyJ..." }

// Response
{
  "success": true,
  "total_users": 1500,
  "total_articles": 240,
  "total_categories": 12,
  "total_views": 48200,
  "articles_today": 3,
  "users_today": 17,
  "total_banned": 4
}
ListUsers admin

Returns a paginated, filterable list of all platform users. Filter by role (0=user, 1=admin, -1=all) or status (0=active, 1=banned, -1=all).

Request
FieldTypeRequiredDescription
access_tokenstringyes
pageintnoDefault 1
limitintnoDefault 20
searchstringnoSearch by email or username
roleintno-1=all, 0=user, 1=admin
statusintno-1=all, 0=active, 1=banned
Response
FieldTypeDescription
successbool
usersUserRecord[]id, email, username, role, status, created_at, banned_at, ban_reason
pagination.pageint
pagination.totalint
pagination.total_pagesint
json
// Request
{ "access_token": "eyJ...", "page": 1, "limit": 20, "search": "john", "role": -1, "status": -1 }

// Response
{
  "success": true,
  "users": [
    { "id": "42", "email": "john@example.com", "username": "john_doe", "role": 0, "status": 0, "created_at": "2024-01-15T10:00:00Z" }
  ],
  "pagination": { "page": 1, "limit": 20, "total": 1, "total_pages": 1 }
}

GetUploadUrl admin

Returns a pre-signed PUT URL for uploading article cover images directly from the browser to object storage. PUT the image binary to upload_url, then use file_url as the article's cover_image_url.

Request
FieldTypeRequiredDescription
access_tokenstringyes
filenamestringyese.g. "cover.jpg"
content_typestringyese.g. "image/jpeg"
article_idstringnoAssociate with existing article
Response
FieldTypeDescription
upload_urlstringPre-signed PUT URL — upload binary here
file_urlstringPublic URL — use as cover_image_url
expires_inintSeconds until upload_url expires
json
// Request
{ "access_token": "eyJ...", "filename": "cover.jpg", "content_type": "image/jpeg" }

// Response
{ "success": true, "upload_url": "https://s3.../presigned...", "file_url": "https://cdn.../cover.jpg", "expires_in": 300 }

CreatePromoCode admin

Creates a discount promo code (proxied to subscription-service). discount_type is "percentage" (1–100) or "fixed" (cents).

Request
FieldTypeRequiredDescription
access_tokenstringyes
codestringyesUnique code string
discount_typestringyes"percentage" or "fixed"
discount_valueintyesPercentage (1–100) or fixed cents
max_usesintno0 = unlimited
applicable_tiersint[]noEmpty = all tiers
min_duration_monthsintnoMinimum purchase duration
valid_untilstringnoISO 8601, empty = no expiry
json
// Request
{
  "access_token": "eyJ...",
  "code": "SAVE20",
  "discount_type": "percentage",
  "discount_value": 20,
  "max_uses": 100,
  "applicable_tiers": [2, 3],
  "valid_until": "2026-12-31T00:00:00Z"
}

// Response
{ "success": true, "promo_code": { "id": "pc_1", "code": "SAVE20", "discount_type": "percentage", "discount_value": 20, "active": true } }

AdminSendBulkNotification admin

Sends the same notification to multiple recipients at once. Response includes delivery counts for sent and failed messages (proxied to notification-service).

Request
FieldTypeRequiredDescription
access_tokenstringyes
subjectstringyes
bodystringyes
channelstringno"email" or "in_app"
recipients{ user_id, email }[]yes
Response
FieldTypeDescription
successbool
totalintTotal recipients
sentintSuccessfully delivered
failedintFailed deliveries
json
// Request
{
  "access_token": "eyJ...",
  "subject": "Platform Maintenance",
  "body": "We will be down for maintenance on April 5.",
  "channel": "email",

  ]
}

// Response
{ "success": true, "message": "Bulk notification sent", "total": 2, "sent": 2, "failed": 0 }
