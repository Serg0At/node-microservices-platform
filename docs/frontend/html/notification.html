Notification Service API Reference

Delivers transactional emails triggered by auth events (registration, login alerts, password changes, 2FA) and manages an in-app notification inbox that users can read, mark as read, and delete.

All RPCs are exposed through the GraphQL gateway. user_id is typically resolved from the JWT by the gateway.
GetNotifications @auth

Returns a paginated list of notifications for the current user. Filter by type_filter (e.g. "new_login") or read_filter ("read", "unread", or "" for all).

Request
FieldTypeRequiredDescription
user_idstringyes
limitintnoDefault 20, max 100
offsetintnoDefault 0
type_filterstringnoe.g. "welcome", "verify_email", "new_login"
read_filterstringno"read", "unread", or "" for all
Response — Notification fields
FieldTypeDescription
successbool
total_countint64Total matching notifications
notifications[].idstring
notifications[].typestringe.g. new_login, welcome
notifications[].channelstringemail, in_app, push
notifications[].titlestring
notifications[].bodystring
notifications[].statusstringpending, sent, failed, read
notifications[].readbool
notifications[].created_atstringISO 8601
notifications[].read_atstringISO 8601, null if unread
json
// Request
{ "user_id": "42", "limit": 20, "read_filter": "unread" }

// Response
{
  "success": true,
  "total_count": 2,
  "notifications": [
    {
      "id": "notif_1",
      "type": "new_login",
      "title": "New login detected",
      "body": "A new login was detected from Chrome on Windows.",
      "read": false,
      "created_at": "2026-04-01T09:00:00Z"
    }
  ]
}

Request
FieldTypeRequired
notification_idstringyes
user_idstringyes
json
