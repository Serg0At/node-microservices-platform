Subscription Service API Reference

Manages the full subscription lifecycle including plan checkout, cancellation, restoration, and promo code validation. Publishes events to RabbitMQ on every subscription state change. All routes require a JWT access token.

All RPCs are exposed through the GraphQL gateway. sub_type: 0=None, 1=Lite, 2=Standard, 3=PRO.
GetSubscription @auth

Returns the current user's active subscription details including tier, status, and access dates. Check grace_period_end — users retain access until this date even after expiry.

Request
FieldTypeRequired
access_tokenstringyes
Response
FieldTypeDescription
successbool
subscription.idstring
subscription.sub_typeint0–3
subscription.free_trialboolWhether this is a trial
subscription.statusstringactive, canceled, expired, terminated
subscription.started_atstringISO 8601
subscription.ended_atstringISO 8601
subscription.grace_period_endstringISO 8601 — access retained until here
subscription.issued_bystring"User", "Admin", "Promo"
json
// Request
{ "access_token": "eyJ..." }

// Response
{
  "success": true,
  "subscription": {
    "id": "sub_1",
    "user_id": "42",
    "sub_type": 3,
    "free_trial": false,
    "status": "active",
    "started_at": "2026-01-01T00:00:00Z",
    "ended_at": "2027-01-01T00:00:00Z"
  }
}
CheckAccess @auth

Checks whether the user has at least the required subscription level. Used internally by the gateway for feature gating — you can also call this directly to conditionally show/hide premium features.

Request
FieldTypeRequiredDescription
user_idstringyes
required_levelintyesMinimum sub_type needed
Response
FieldTypeDescription
has_accessbooltrue if current_level >= required_level
current_levelintUser's actual sub_type
statusstringSubscription status
json
// Request
{ "user_id": "42", "required_level": 2 }

// Response
{ "has_access": true, "current_level": 3, "status": "active" }
CreateCheckout @auth

Initiates a payment checkout for a subscription plan. Redirect the user to payment_url to complete payment. The response also includes proration details if the user is upgrading an existing plan.

Request
FieldTypeRequiredDescription
access_tokenstringyes
plan_typeintyes1=Lite, 2=Standard, 3=PRO
payment_methodstringyes"crypto" or "card"
duration_monthsintyes1, 3, 6, or 12
promo_codestringnoOptional discount code
Response
FieldTypeDescription
payment_urlstringRedirect user here to complete payment
order_idstringInternal order reference
expires_inintSeconds until checkout session expires
proration.remaining_daysintDays left on current plan
proration.remaining_value_centsintCredit from current plan
proration.new_plan_price_centsintFull price of new plan
proration.discount_centsintPromo discount applied
proration.final_price_centsintAmount user will be charged
json
// Request
{ "access_token": "eyJ...", "plan_type": 3, "payment_method": "crypto", "duration_months": 12, "promo_code": "SAVE20" }

// Response
{
  "success": true,
  "payment_url": "https://pay.cryptomus.com/...",
  "order_id": "ord_abc123",
  "expires_in": 1800,
  "proration": {
    "remaining_days": 15,
    "remaining_value_cents": 500,
    "new_plan_price_cents": 9900,
    "discount_cents": 1980,
    "final_price_cents": 7420
  }
}
CancelSubscription @auth

Cancels the current user's subscription at period end. The user retains access until ended_at and can restore before then using RestoreSubscription.

Request
FieldTypeRequired
access_tokenstringyes
json
// Request
{ "access_token": "eyJ..." }

// Response
{ "success": true, "subscription": { "id": "sub_1", "status": "canceled", "ended_at": "2027-01-01T00:00:00Z" } }
RestoreSubscription @auth

Restores a previously canceled subscription before it expires, setting status back to active.

Request
FieldTypeRequired
access_tokenstringyes
json
// Request
{ "access_token": "eyJ..." }

// Response
{ "success": true, "subscription": { "id": "sub_1", "status": "active" } }
ValidatePromoCode @auth

Validates a promo code for a given plan and returns the calculated discount without creating a checkout — ideal for live discount previews in the UI.

Request
FieldTypeRequired
access_tokenstringyes
codestringyes
plan_typeintyes
duration_monthsintyes
Response
FieldTypeDescription
validboolWhether the code is applicable
discount_typestring"percentage" or "fixed"
discount_valueint
discount_amount_centsintCalculated saving in cents
final_price_centsintPrice after discount
messagestringError reason if valid: false
json
// Request
{ "access_token": "eyJ...", "code": "SAVE20", "plan_type": 3, "duration_months": 12 }

// Response
{
  "success": true,
  "valid": true,
  "discount_type": "percentage",
  "discount_value": 20,
  "discount_amount_cents": 1980,
  "final_price_cents": 7920
