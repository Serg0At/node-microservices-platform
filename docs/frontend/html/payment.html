Payment Service API Reference

Handles payment initiation and transaction history for crypto (Cryptomus) and card (Fondy) providers. Webhook endpoints are called directly by payment providers and are not accessible from the frontend.

All RPCs are exposed through the GraphQL gateway. Webhook endpoints (/webhook/cryptomus, /webhook/fondy) are provider-facing — not for frontend use.
CreatePayment @auth

Creates a payment order and returns a provider URL to redirect the user to. In practice, the gateway calls this on your behalf when you call CreateCheckout on the subscription-service — you rarely call this directly.

Request
FieldTypeRequiredDescription
user_idstringyes
plan_typeintyes1=Lite, 2=Standard, 3=PRO
payment_methodstringyes"crypto" or "card"
currencystringyese.g. "USD", "USDT"
amountintyesAmount in cents
order_idstringyesUnique order reference from subscription-service
duration_monthsintyes1, 3, 6, or 12
Response
FieldTypeDescription
successbool
payment_urlstringRedirect user here to complete payment
order_idstringConfirmed order ID
expires_inintSeconds until payment session expires
json
// Request
{
  "user_id": "42",
  "plan_type": 3,
  "payment_method": "crypto",
  "currency": "USDT",
  "amount": 9900,
  "order_id": "ord_abc123",
  "duration_months": 12
}

// Response
{
  "success": true,
  "payment_url": "https://pay.cryptomus.com/pay/...",
  "order_id": "ord_abc123",
  "expires_in": 1800
}
GetTransaction @auth

Returns full details of a single transaction by ID. Use this to show a receipt or check payment status after redirect.

Request
FieldTypeRequired
access_tokenstringyes
idstringyes — transaction ID
Response
FieldTypeDescription
transaction.idstring
transaction.providerstring"cryptomus" or "fondy"
transaction.amountintAmount in cents
transaction.currencystring
transaction.statusstringpending, paid, failed, refunded
transaction.plan_typeint
transaction.duration_monthsint
transaction.created_atstringISO 8601
json
// Request
{ "access_token": "eyJ...", "id": "txn_xyz" }

// Response
{
  "success": true,
  "transaction": {
    "id": "txn_xyz",
    "user_id": "42",
    "provider": "cryptomus",
    "amount": 9900,
    "currency": "USDT",
    "status": "paid",
    "plan_type": 3,
    "duration_months": 12
  }
}
ListTransactions @auth

Returns a paginated list of transactions. Filter by status to show only paid invoices. Admins can also filter by user_id to pull another user's transactions.

Request
FieldTypeRequiredDescription
access_tokenstringyes
user_idstringnoFilter by user (admin use)
statusstringnopending, paid, failed, refunded
pageintnoDefault 1
limitintnoDefault 20
json
// Request
{ "access_token": "eyJ...", "status": "paid", "page": 1, "limit": 10 }

// Response
{
  "success": true,
  "transactions": [
    { "id": "txn_xyz", "amount": 9900, "currency": "USDT", "status": "paid", "provider": "cryptomus" }
  ],
  "total": 1
}
