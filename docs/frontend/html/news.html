News Service API Reference

Stores and serves articles (blogs and news), supports full-text search and category filtering, and provides presigned upload URLs for cover images. Read operations are public; write operations require an admin JWT.

All RPCs are exposed through the GraphQL gateway. Read operations (GetArticle, ListArticles, SearchArticles) are public — no token needed.
GetArticle public

Returns the full content of a single article. Look it up by id or slug — provide one or the other.

Request
FieldTypeRequiredDescription
idstringone ofArticle ID
slugstringone ofURL-friendly slug
Response
FieldTypeDescription
article.idstring
article.titlestring
article.slugstring
article.contentstringFull article body
article.author_idstring
article.typestring"blog" or "news"
article.categoriesstring[]
article.cover_image_urlstring
article.view_countint64
article.published_atstringISO 8601
json
// Request
{ "slug": "market-update-april" }

// Response
{
  "success": true,
  "article": {
    "id": "101",
    "title": "Market Update April",
    "slug": "market-update-april",
    "content": "...",
    "type": "news",
    "categories": ["crypto"],
    "view_count": 1240,
    "published_at": "2026-04-01T08:00:00Z"
  }
}
ListArticles public

Returns a paginated list of article summaries with optional category and author filters. Use this for article feeds and index pages.

Request
FieldTypeRequiredDescription
pageintnoDefault 1
limitintnoDefault 10, max 50
categorystringnoFilter by category name
author_idstringnoFilter by author
Response
FieldTypeDescription
articlesArticleSummary[]id, title, slug, author_id, type, categories[], cover_image_url, view_count, published_at
pagination.pageint
pagination.totalint
pagination.total_pagesint
json
// Request
{ "page": 1, "limit": 10, "category": "crypto" }

// Response
{
  "success": true,
  "articles": [{ "id": "101", "title": "Market Update April", "slug": "market-update-april", "type": "news", "view_count": 1240 }],
  "pagination": { "page": 1, "limit": 10, "total": 42, "total_pages": 5 }
}

CreateArticle admin

Creates a new article. Use GetUploadUrl first to upload a cover image and obtain a public URL, then pass it as cover_image_url.

Request
FieldTypeRequiredDescription
access_tokenstringyesAdmin JWT
titlestringyes
contentstringyesFull article body
typestringyes"blog" or "news"
cover_image_urlstringnoURL from GetUploadUrl
categoriesstring[]no
json
// Request
{ "access_token": "eyJ...", "title": "Bitcoin Halving 2024", "content": "...", "type": "news", "categories": ["bitcoin", "halving"] }

// Response
{ "success": true, "article": { "id": "102", "title": "Bitcoin Halving 2024", "slug": "bitcoin-halving-2024", "type": "news" } }

GetUploadUrl admin

Returns a pre-signed PUT URL for uploading a cover image directly from the browser. PUT the binary to upload_url, then use file_url as the article's cover_image_url. The upload_url expires after expires_in seconds.

Request
FieldTypeRequiredDescription
access_tokenstringyes
filenamestringyese.g. "cover.jpg"
content_typestringyese.g. "image/jpeg"
article_idstringnoAssociate with an existing article
Response
FieldTypeDescription
upload_urlstringPUT image binary here — expires in expires_in seconds
file_urlstringPublic URL to use as cover_image_url
expires_inintSeconds
json
// Request
{ "access_token": "eyJ...", "filename": "cover.jpg", "content_type": "image/jpeg" }

// Response
{ "success": true, "upload_url": "https://s3.../presigned...", "file_url": "https://cdn.../cover.jpg", "expires_in": 300 }
